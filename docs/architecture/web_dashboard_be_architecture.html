주요 요청 흐름

AUTH 로그인 흐름

  • 1사용자가 /web_login.html에서 ID/PW 입력
  • 2POST /api/v1/auth/login → Nginx 프록시
  • 3auth/router.py → auth/service.py:login_with_password()
  • 4auth/db/crud.py → users 테이블 조회 · bcrypt 검증
  • 5auth/jwt.py → access_token(15m) + refresh_token(7d) 발급
  • 6FE: sessionStorage.sfp_access_token + localStorage.dashboard_auth_v1 저장
  • 7window.location → /web_dashboard.html 이동

DASHBOARD 대시보드 데이터 로딩

  • 1dashboard_app.js 탭 활성화 → DASHBOARD_API.fetchBundle(tab)
  • 2dashboard_api.js:authorizedFetch() → Authorization: Bearer {token}
  • 3Nginx → GET /api/v1/dashboard/web/{tab} → FastAPI
  • 4dashboard/router.py — get_current_user Depends로 JWT 검증
  • 5dashboard/service.py:get_web_{tab}_dashboard() 호출
  • 6dashboard/repository.py:get_web_dashboard_snapshot() — 다수 SQL 쿼리 실행
  • 7KPI 계산 · OEE · risk_score → JSON 번들 조립 후 반환
  • 8dashboard_api.js:normalizeBundle() → dashboard_app.js:renderAll()

TOKEN 토큰 자동 갱신 (401 intercept)

  • 1API 응답 401 수신
  • 2dashboard_api.js:refreshAccessToken() 호출
  • 3POST /api/v1/auth/refresh with refresh_token
  • 4auth/service.py:refresh_access_token() → 새 access_token 발급
  • 5sessionStorage.sfp_access_token 갱신
  • 6원래 요청 재시도 (새 토큰으로)
  • 7재시도도 401이면 redirectToLogin() → 로그아웃

REPOSITORY WebDashboardSnapshot 쿼리 구조

  • 1_dashboard_time_bounds() — shift_start / effective_now / elapsed_ratio (KST)
  • 2production_records — total / month 생산량 합산
  • 3alarms → active/hold 알람 수 + 상세 목록 (JOIN lines, equipments)
  • 4defect_results → top_defects (상위 6종)
  • 5lines → 라인별 OEE 계산 (availability × performance × quality)
  • 6_resolve_web_focus_line() — 포커스 라인 결정 (요청 → LINE-C → risk_score 최악)
  • 7recheck_queue / inspection_results / event_logs / line_environment
  • 810분 버킷 NG 트렌드 · 설비 가동 상태 → WebDashboardSnapshot 반환
