### RAG 검색 전략

```
1. Dense Retrieval   → FAISS 벡터 유사도 (text-embedding-3-small)
2. Sparse Retrieval  → 토큰 오버랩 스코어링 (FAISS 실패 시 폴백)
3. 결과 병합         → 중복 제거 + 관련도 순 정렬
```

### 웹 검색 폴백 순서

```
Serper (뉴스) → Serper (텍스트) → Tavily → OpenAI Web Search
```

## 3. Database

### PostgreSQL 16

| 항목 | 값 |
|------|----|
| 이미지 | `postgres:16-alpine` |
| DB명 | `Final_Project_DB` |
| 포트 | `40919:5432` |
| ORM | SQLAlchemy 2.0 + psycopg 3 |
| 스키마 파일 | `server_api/sql/wed_dashboard_schema.sql` |

**테이블 계층 구조**

```
Factory
  └── Line
        └── Equipment
              └── Employee
```

**주요 테이블**

| 그룹 | 테이블 | 비고 |
|------|--------|------|
| 마스터 | FactoryTable, LineTable, EquipmentTable, EmployeeTable | 계층 FK 구조 |
| 운영 | ProductionRecordsTable | lot_id, 생산량, 불량수, 교대 |
| 설비 | EquipmentStatusHistoryTable | run / idle / down / maint |
| 품질 | 불량 분류 테이블 | short, open, spur, mouse_bite, spurious_copper |
| 인증 | UserTable, FaceEmbeddingTable, VectorTable | TSVECTOR (얼굴 임베딩 검색) |

**특수 컬럼 타입**

| 타입 | 용도 |
|------|------|
| `TIMESTAMPTZ` | 시계열 데이터 (시간대 포함) |
| `TSVECTOR` | Face Embedding 빠른 조회 |
| `JSON` | 임베딩 원본 저장 |

|------|----|
| 이미지 | `redis:7.4-alpine` |
| 포트 | `40917:6379` |

**포트 맵**

| 서비스 | 외부 포트 | 내부 포트 |
|--------|----------|----------|
| Nginx HTTP | 40916 | 80 |
| Nginx HTTPS | 40943 | 443 |

| PostgreSQL | 40919 | 5432 |
| Redis | 40917 | 6379 |

**Nginx 주요 설정**

- HTTP(80) → HTTPS(443) 리다이렉트
- `/api/*` → `proxy_pass http://server_api:8000`
- HTML / JS / CSS: `Cache-Control: no-store, no-cache`
- 404 폴백: `login.html`
