# QA_FE_구조개선_작업기준서

## 1. 문서 목적

본 문서는 기존 FE의 `QA` 화면을 역할 기반 구조로 재편하기 위한 작업 기준서이다.  
목적은 다음과 같다.

- QA 화면의 1차 정보 구조를 `원인 파악 + 재검 우선순위 확정` 중심으로 재정의한다.
- 첫 화면에 남길 정보와 상세 팝업으로 분리할 정보를 명확히 구분한다.
- 메인 화면과 상세 팝업의 역할, 인터랙션 방식, 구현 영향 범위를 FE 관점에서 정리한다.
- 이후 화면 디자인 및 실제 FE 구조 개선 작업 시 Codex와 작업자가 계속 참조할 수 있는 기준 문서로 사용한다.

## 3. QA 화면 핵심 역할 재정의

| 항목 | 내용 |
|---|---|
| 주요 사용자 | 품질 엔지니어, 검사 리더 |
| 한 줄 목적 | 불량 급증 원인과 재검 우선순위를 빠르게 확정 |
| 첫 판단 질문 | `불량이 어디서 왜 늘었고, 어떤 Lot/검사를 먼저 재확인해야 하는가?` |
| 화면 성격 | 분석형 + 즉시 대응형 |
| 개편 핵심 원칙 | `원인 Top + 재검 우선순위` 중심, 상세 추적은 클릭 후 확인 |

### 역할 해석
QA 화면은 단순 현황판이 아니라 `원인 판단`과 `재검 실행 우선순위`를 동시에 지원해야 한다.  
다만 첫 화면에서 모든 추적 정보와 근거 데이터를 노출하면 과밀해지므로, 1차 판단 정보와 2차 확인 정보를 반드시 분리해야 한다.

## 4. 첫 화면 고정 원칙

### 4-1. 첫 화면에 반드시 남길 정보
- 현재 불량률(임계치 대비)
- 재검 필요 수량
- 주요 불량 원인 Top3
- 재검 우선 큐 Top3
- 원인 기여도 요약
- 불량률 경향 요약
- 품질 리스크/경보
- 조건부 패턴 힌트

### 4-2. 첫 화면 구성 원칙
- 카드/블록 수는 5~7개 이내로 유지
- 차트는 최대 1~2개
- 대형 테이블은 첫 화면에서 제거
- 긴 텍스트 대신 상태 배지, Top 막대, 요약 카드, 우선순위 리스트로 치환
- `좌상단(품질 상태) → 중앙(원인 판단) → 우측(행동 대상) → 하단(보조 추세/경보)` 시선 흐름 유지
- `1920×1080` 기준 무스크롤 우선
- `1366×768`에서도 핵심 블록 우선 인지가 가능해야 함

### 4-3. 첫 화면에서 줄여야 할 것
- NG 상세 테이블 전체
- Lot 추적 전체 목록
- 장기 추세 차트
- 판정 근거 원문/이미지 다중 노출
- 재검 대상 Top5 이상 장목록
- 이중 분석 차트 병렬 노출

## 5. 상세 팝업 분리 원칙

### 메인 화면에서 상세로 내려야 할 정보
- Lot 추적 전체 이력
- NG 상세 리스트 전체
- 원인코드 drilldown(라인/설비/시간대)
- 판정 근거/이미지
- 재검 담당자별 SLA
- 검사/판정 이력 전체
- 전체 품질 이슈 목록
- 장기 추세 및 원본 데이터

### 분리 원칙
- 메인 화면은 `판단용`
- 상세 팝업은 `해석 + 실행 연결용`
- 장문/원문/전체 목록은 메인 팝업 본문이 아니라 2차 상세 또는 내부 슬롯으로 처리

## 6. 메인 QA 화면 구조 기준

### 최종 메인 구조
- 상단: 품질 KPI 3개
- 중앙: 주요 불량 원인 Top3 / 기여도
- 우측: 재검 우선 큐 Top3
- 하단 좌측: 불량률 경향 요약
- 하단 우측: 품질 리스크/경보
- 하단 또는 보조: 조건부 패턴 힌트 배너

### 메인 구조 표

| 영역명 | 화면 위치 | 1차 노출 정보 | 목적 | 클릭 여부 | 상세 방식 |
|---|---|---|---|---|---|
| 현재 불량률 KPI | 상단 | 현재값, 상태 | 현재 품질 위험 수준 판단 | 클릭 | 중앙 중형 모달 |
| 재검 필요 수량 KPI | 상단 | 재검 총량, 상태 | 재검 부담 수준 판단 | 클릭 | 중앙 중형 모달 |
| 검사 현황 요약 KPI | 상단 | 총검사/OK/NG | 검사 흐름 흔들림 판단 | 클릭 | 중앙 중형 모달 |
| 주요 불량 원인 Top3 | 중앙 | 원인 Top3, 기여율 | 원인 파악 | 클릭 | 중앙 대형 모달 |
| 원인 기여도 요약 | 중앙 | 집중도/분산도 | 문제 집중 여부 판단 | 클릭 | 중앙 중형 모달 |
| 재검 우선 큐 Top3 | 우측 | 우선순위 Lot/Inspection 3건 | 즉시 재검 대상 확정 | 클릭 | 중앙 대형 모달 |
| 불량률 경향 요약 | 하단 좌측 | 최근 6구간 추세 | 일시 상승/확대 여부 판단 | 클릭 | 중앙 중형 모달 |
| 품질 리스크/경보 | 하단 우측 | ALERT/HOLD/RECHECK 요약 | 현재 이슈 확인 | 클릭 | 우측 슬라이드 패널 |
| 패턴 힌트 배너 | 하단 보조 | 반복 패턴, 신뢰도 | 잠재 징후 확인 | 조건부 클릭 | 소형 오버레이 |

### 메인 구조 결정 사항
- `재검 우선 큐`는 무조건 `Top3` 기준으로 유지
- Top5 이상 확장은 메인 화면 금지
- `원인 Top3 + 재검 Top3` 구도가 화면 핵심
- 검사 현황 요약은 KPI 카드로만 압축 유지
- 경향 요약은 `소형 추세` 수준으로 제한

## 7. 상세 팝업 구조 기준

### 7-1. 상세 화면 목록

| 상세 화면명 | 연결되는 메인 영역 | 팝업 형태 | 목적 |
|---|---|---|---|
| 품질 KPI 상세 모달 | 현재 불량률 KPI / 재검 필요 수량 KPI / 검사 현황 요약 KPI | 중앙 중형 모달 | KPI의 기준 초과 여부와 최근 변화 해석 |
| 원인코드 Drilldown 모달 | 주요 불량 원인 Top3 / 원인 기여도 요약 | 중앙 대형 모달 | 원인별 라인/설비/시간대 편중 확인 |
| 재검 Lot/Inspection 상세 모달 | 재검 우선 큐 Top3 | 중앙 대형 모달 | 이 건을 왜 먼저 재검해야 하는지와 후속 조치 확정 |
| 불량률 추세 상세 모달 | 불량률 경향 요약 | 중앙 중형 모달 | 불량률 상승이 일시적인지 확대 중인지 판단 |
| 품질 이슈 상세 슬라이드 패널 | 품질 리스크/경보 | 우측 슬라이드 패널 | 현재 품질 이슈를 맥락 유지한 채 빠르게 확인 |
| 패턴 힌트 오버레이 | 조건부 패턴 힌트 배너 | 소형 오버레이 | 확정 경보 전 반복 패턴 근거 확인 |

### 7-2. 공통 구조 원칙
모든 상세 팝업은 아래 4층 구조를 따른다.

1. 상단 고정 영역  
   - 제목
   - 대상(라인/설비/Lot/Inspection/KPI/원인)
   - 상태
   - 핵심 수치 또는 식별자
   - 닫기/뒤로
   - 핵심 액션 버튼

2. 본문 핵심 영역  
   - 최근 변화
   - 핵심 차트
   - 핵심 리스트
   - 체크리스트
   - 요약 시각화

3. 하단 상세 영역  
   - 아코디언
   - 탭
   - 내부 스크롤 슬롯

4. 액션 바  
   - 관련 화면 이동
   - Lot 보기
   - 판정 근거 보기
   - 담당 전달
   - 닫기

## 8. 인터랙션 방식 기준

### 8-1. 하단 상세영역 인터랙션 정책

| 방식 | 언제 쓰는가 | 장점 | 단점 | QA 상세 팝업에서 적합한 정보 |
|---|---|---|---|---|
| 고정 | 몇 초 안에 바로 읽고 판단해야 할 때 | 즉시 대응성 높음 | 본문 압박 가능 | 현재 상태, 임계값, 우선순위, 판정 보류 사유 |
| 스크롤 | 길이가 가변적이고 같은 맥락으로 연속 확인해야 할 때 | 많은 항목 수용 가능 | 남발 시 피로도 증가 | Lot 이력, 판정 이력, 연관 이슈, 관련 Lot 목록 |
| 펼침형(아코디언) | 필요 시만 보는 참고/근거 정보일 때 | 기본 화면 가볍게 유지 | 많으면 복잡해짐 | 기준 설명, 오탐 주의, 연관 원인, 담당자/SLA |
| 탭 | 서로 다른 관점의 정보를 전환해서 봐야 할 때 | 주제 구분 명확 | 많으면 맥락 단절 | 라인/설비/시간대 분석, 요약/근거 전환 |

| 팝업명 | 하단 정보 항목 | 방식 | 이유 |
|---|---|---|---|
| 품질 KPI 상세 모달 | 전 근무조 대비 | 펼침형 | 보조 해석 정보 |
| 품질 KPI 상세 모달 | 기준 설명 | 펼침형 | 자주 보지 않는 설명성 정보 |
| 품질 KPI 상세 모달 | 관련 Lot 진입 버튼 | 고정 | 행동 직결 정보 |
| 원인코드 Drilldown 모달 | 라인/설비/시간대 | 탭 | 분석 축이 다름 |

| 원인코드 Drilldown 모달 | 이미지/판정 근거 진입 | 펼침형 | 필요 시만 확인 |

| 재검 Lot/Inspection 상세 모달 | Lot 추적 진입 | 고정 | 핵심 후속 행동 |
| 재검 Lot/Inspection 상세 모달 | 담당자/SLA | 펼침형 | 항상 먼저 볼 정보는 아님 |
| 재검 Lot/Inspection 상세 모달 | 연관 알람/이슈 | 스크롤 | 길이 가변, 연속 확인 필요 |
| 불량률 추세 상세 모달 | 관련 원인 Top2 | 고정 | 추세 해석 직결 |
| 불량률 추세 상세 모달 | 재검 영향 수량 | 고정 | 대응 필요성 판단 직결 |
| 품질 이슈 상세 패널 | 연관 원인 | 펼침형 | 보조 참고 |
| 품질 이슈 상세 패널 | 관련 Lot | 스크롤 | 케이스별 가변 목록 |
| 품질 이슈 상세 패널 | 담당자/전달 상태 | 펼침형 | 길이가 짧은 보조 상태 |
| 패턴 힌트 오버레이 | 유사 사례 수 | 고정 | 짧고 핵심적인 보조 근거 |
| 패턴 힌트 오버레이 | 오탐 주의 문구 | 펼침형 | 필요 시만 확인 |

### 8-3. 인터랙션 적용 주의사항
- 전체 팝업 세로 스크롤은 기본 금지
- 내부 스크롤은 `목록 슬롯`에만 허용
- 탭은 최대 3개 이내
- 아코디언은 2개 수준 권장
- 긴 로그, 장문 메모, 다중 표/차트는 메인 팝업 본문 금지
- `고정 -> 핵심 판단 -> 조건부 상세` 순서를 깨지 않도록 유지

## 9. 구현 시 주의사항

### 유지할 것
- `원인 Top3 + 재검 우선 큐 Top3` 중심 구조
- KPI 3개 상단 요약
- 하단 경향 요약 + 품질 리스크 이원 구조
- 조건부 패턴 힌트 배너

### 축소할 것
- 재검 큐 Top5 이상 확장
- 대형 불량률 추세 차트
- 검사 상세 테이블 본문 노출
- 판정 근거 원문 직접 노출

### 모달/상세로 내려야 할 것
- Lot 추적 전체
- 원인코드 drilldown 세부 분해
- 이미지/판정 근거
- 담당자/SLA
- 검사/판정 이력
- 장기 추세
- 전체 품질 이슈 목록

### 절대 메인 화면에 다시 올리면 안 되는 것
- NG 상세 테이블 전체
- Lot 추적 테이블 전체
- 원인코드 상세 테이블
- 장기 추세 대형 차트
- Top5 이상 재검 큐
- 이미지 다중 썸네일 리스트
- 장문 설명 블록

### 구현 우선순위 제안
1. 메인 QA 화면 블록 재배치
2. 재검 우선 큐 Top3로 축소 및 카드 구조 통일
3. 원인 Top3 / 기여도 블록 시각화 정리
4. 품질 KPI 상세 모달 구조 정리

## 13. 최종 작업 기준 요약

- 메인 QA 화면은 `원인 Top3 + 재검 Top3` 중심으로 고정한다.
- 1차 화면은 판단용, 상세 팝업은 해석 및 실행 연결용으로 분리한다.
- 하단 상세영역은 정보 성격에 따라 `고정 / 탭 / 아코디언 / 내부 스크롤`을 구분 적용한다.
- `전체 목록`, `원문`, `장기 추세`, `다중 차트`는 메인 화면과 메인 팝업 본문에 두지 않는다.
- 구현 중 구조 충돌이 발생하면 `무스크롤`, `원인 우선`, `재검 Top3`, `팝업 분리` 원칙을 우선 적용한다.
