# 기술 스택 정의서 — Web Dashboard

분석 기준: `nginx/html/web_login.html`, `nginx/html/web_dashboard.html`, `nginx/html/js/web_login/`, `nginx/html/js/wed_dashboard/`, `server_api/src/modules/auth/`, `server_api/src/modules/dashboard/`
작성일: 2026-03-23

---

## 흐름 개요

```
[Browser]
    │
    ├─ GET /web_login.html
    │       └─ web_login.js
    │               ├─ POST /api/v1/auth/login          (ID/PW)
    │               ├─ POST /api/v1/auth/login/face     (Face ID)
    │               └─ GET  /api/v1/auth/web-login-profile/{employee_no}
    │                       └─ 세션 저장 후 /web_dashboard.html 리다이렉트
    │
    └─ GET /web_dashboard.html
            └─ dashboard_app.js (6개 모듈 조합)
                    ├─ GET  /api/v1/dashboard/web/{worker|qa|manager|promo}
                    ├─ GET  /api/v1/dashboard/detail
                    └─ POST /api/v1/auth/refresh  (access token 만료 시)
```

## 1. Frontend

### 파일 구성

| 파일 | 역할 |
|------|------|
| `web_login.html` | 로그인 화면 (ID/PW + Face ID 탭 전환) |
| `web_dashboard.html` | 대시보드 셸 (사이드바 + 필터바 + 콘텐츠 영역) |
| `css/web_login/web_login_style.css` | 로그인 화면 전용 스타일 |
| `css/wed_dashboard/dashboard_style.css` | 대시보드 레이아웃 |
| `css/wed_dashboard/dashboard_status.css` | 상태 배지 (run/idle/down/maint) |

### JavaScript 모듈 로드 순서

```
dashboard_state.js      전역 상태 저장소
dashboard_thresholds.js 임계값 상수 정의
dashboard_api.js        fetch 통신 레이어 (DashboardAPI 객체)
dashboard_detail_views.js 상세 모달 렌더링
dashboard_charts.js     Chart.js 래퍼
dashboard_app.js        위 모듈을 조합해 실제 화면 렌더링
```

### 라이브러리

| 기술 | 버전 | 용도 | 비고 |
|------|------|------|------|
| Chart.js | 4.4.1 | 차트 렌더링 | 로컬 `vendor/chart.umd.min.js` 우선, 실패 시 jsDelivr CDN 폴백 |
| Google Fonts | - | DM Sans, DM Mono, Noto Sans KR | `web_login.html` CDN 로드 |
| Browser Fetch API | - | REST API 통신 | 별도 HTTP 라이브러리 없음 |
| MediaDevices API | - | Face ID 웹캠 스트림 (`getUserMedia`) | HTTPS 필수 (`window.isSecureContext` 검사) |
| Canvas API | - | 웹캠 프레임 캡처 → JPEG Base64 변환 | quality 0.9, max 10 FPS |

## 3. API 엔드포인트

### 인증 (`/api/v1/auth`)

| 메서드 | 경로 | 요청 | 응답 |
|--------|------|------|------|
| POST | `/login` | `{ employee_no, password }` | `LoginResponse` (access + refresh token) |
| POST | `/login/face` | `{ employee_no?, image_base64 }` | `FaceLoginResponse` (verified, access + refresh token) |
| POST | `/refresh` | `{ refresh_token }` | `RefreshTokenResponse` (access token) |
| GET | `/web-login-profile/{employee_no}` | Bearer token | `WebLoginProfileResponse` (name, role) |

> Face ID: `employee_no` 있으면 1:1, 없으면 등록된 전체 대상으로 1:N 비교

### 대시보드 (`/api/v1/dashboard`)

| 메서드 | 경로 | 공통 쿼리 파라미터 | 응답 |
|--------|------|--------------------|------|
| GET | `/web/worker` | `tz, factory, line, shift, period, date_from, date_to` | `WorkerWebDashboardResponse` |
| GET | `/web/qa` | 동일 | `QaWebDashboardResponse` |
| GET | `/web/manager` | 동일 | `ManagerWebDashboardResponse` |
| GET | `/web/promo` | 동일 | `PromoWebDashboardResponse` |
| GET | `/kpis` | `screen, tz, factory, line, shift, period` | `DashboardKPIResponse` |
| GET | `/datasets` | 동일 | `DashboardDatasetsResponse` |
| GET | `/detail` | `screen, detailId, targetType, targetId, subKey` | `DashboardDetailResponse` |

> `worker` 계정은 `service` 계층에서 `line` 필터를 현재 사용자의 라인으로 강제 적용

## 4. Database

### PostgreSQL 16

**주요 테이블 (dashboard/db/model.py 기준)**

```
FactoryTable  (factories)
  └── LineTable  (lines)           FK: factory_id, INDEX: factory_id, is_active
        └── EquipmentTable         FK: line_id
              └── EmployeeTable    FK: line_id
```

**운영 데이터 테이블**

| 테이블 | 주요 컬럼 |
|--------|----------|
| ProductionRecords | lot_id, produced_qty, good_qty, ng_qty, work_shift |
| EquipmentStatusHistory | status_code (run/idle/down/maint), duration |

**컬럼 타입 특이사항**

| 타입 | 용도 |
|------|------|
| `DateTime(timezone=True)` | 모든 시각 컬럼 (`server_default: NOW()`) |
| `TSVECTOR` | Face Embedding 벡터 검색 (VectorTable) |

**인증 테이블 (UserTable 주요 컬럼)**

| 컬럼 | 역할 |
|------|------|
| `employee_no` | PK 역할의 로그인 식별자 |
| `token_version` | JWT 무효화 버전 카운터 |
| `id_active` | 계정 활성화 여부 |
| `id_locked` | 잠금 상태 (로그인 실패 횟수 초과) |
| `login_fail_count` | 연속 실패 횟수 추적 |
