# RAG 모듈 운영 가이드

## 1. 한 줄 설명

RAG(검색 증강 생성) 관련 상태 점검용 모듈입니다.

## 2. API 기본 경로

- `/api/v1/rag`

## 3. 주요 기능

- RAG 모듈 상태 확인

## 4. 주요 엔드포인트

- `GET /api/v1/rag/status`

## 5. 운영 점검 포인트

1. 상태 응답이 살아 있어야 상위 LLM 통합에서 연계 가능
2. 장애 시 LLM 모듈과 함께 로그를 확인

## 6. 자주 발생하는 문제

- 증상: 기능은 정상인데 상태만 점검 필요
- 조치: status 호출 후 장애 여부를 1차 판단
